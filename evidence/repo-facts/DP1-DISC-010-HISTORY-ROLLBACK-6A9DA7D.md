## Factual question

Which tracked prior states, historical regions and rollback inputs are observable for the handoff candidate and its existing workflows?

- `docs/handoff/CURRENT_HANDOFF.md`
- `.agentic/handoff_state.yaml`

- Git-tracked history of the candidate and source files up to the validation ref

## Observed result

1. Git history retains prior committed states of `CURRENT_HANDOFF.md`, its machine-readable state inputs and the relevant generator/writer code.
2. `CURRENT_HANDOFF.md` contains accumulated historical prose outside the marked current operational handoff block.
3. `.agentic/operational_handoff_state.yaml` is an observed source for the generated current-state block; it is not a complete source for all historical prose in the document.
4. The existing block-replacement primitive preserves all bytes outside exactly one marked block and rejects missing, duplicated or misordered markers.
5. The active administrative refresh path also appends or replaces standardized refresh-history sections, so prior document shape is not reconstructed solely from the operational state source.
6. Git checkout/revert of prior commits is an observable source of prior complete-file and source-file bytes.
7. No separate canonical history database or DPA-specific rollback store was observed.
8. No function was observed that automatically merges historical prose from divergent source and target states.
9. Existing administrative branch workflows can reset a refresh branch to `main`, but that repository-operation recovery is not a semantic rollback contract for projected document content.
10. Recovering only the generated current-state block can use declared state plus the renderer; recovering the surrounding historical/manual bytes depends on preserved target bytes or Git history.

## Limitations and unresolved questions

- This record does not select full, hybrid or managed-head form for `CURRENT_HANDOFF.md`.
- It does not prove that every required rollback input remains available after future migration.
- It does not define retention duration or repository-history policy.
- Exact rollback behavior for partial writes, marker corruption, competing PRs and direct writes remains a Probe and DPA-700 obligation.
